# Human-Computer Interaction Project <br> ECFCA System – Registry module and academic tracking 

# 1. User Group Selection

Selected Group: Graduates and external professionals wishing to take
diploma courses or continuing education.

This group requires clear, fast, and accessible processes because they
combine study with work.

# 2. User Description (Persona) Example

Name: Monica -- Professional Learning new skills

Age: 30--45 years

Profile: Active professional seeking certification.

Technology level: Medium--High.

Main need: To register easily and access content without
complications.

Main frustration: Confusing processes or lack of confirmation.

# 3. Functional Requirements

  -----------------------------------------------------------------------
  ID                                  Description
  ----------------------------------- -----------------------------------
  FR-01                               Document upload in PDF, JPG, or
                                      PNG format.

  FR-02                               Automatic validation of the file
                                      format.

  FR-03                               Document receipt notification.

  FR-04                               Administrative validation of the
                                      registration.

  FR-05                               Final confirmation to the user.

  FR-06                               Registration status inquiry.

  FR-07                               Group registration of participants.
  -----------------------------------------------------------------------

# 4. Non-Functional Requirements

  -----------------------------------------------------------------------
  Type                                Requirement
  ----------------------------------- -----------------------------------
  Usability                           Complete registration in less than 5
                                      minutes.

  Availability                        System accessible 24/7.

  Security                            Data protection through
                                      authentication.

  Performance                         Page load time less than 3
                                      seconds.

  Compatibility                       Works on PC, tablet, and mobile
                                      phone.

  Scalability                         Supports multiple simultaneous
                                      users.

  Integrity                           No data loss in case of failures.
  -----------------------------------------------------------------------

# 5. Usability Requirements

Visibility of the process status through clear indicators.

Error prevention with validations before submission.

Simple, non-technical language.

Navigation in a maximum of three steps.

Immediate feedback after each action.

Consistent and clean design to reduce cognitive load.

# 6. Assignment of Responsibilities

UX/UI: User-centered design.

Frontend: Interface implementation.

Backend: File management and logic.

Administrator: Registration validation.

QA: Usability testing.

Security: Data protection.

# 7. Team Reflections

Digitalization requires understanding the user, not just programming.

Usability is key to system adoption.

The design must adapt to professionals with limited time.

HCI allows us to create systems that are not only functional but also human.
